        Все настройки храним в папке профиля пользовтаеля (.poker). Что там будет:
            Собственно файл настроек (темы, цвета, оформление, настройки интерфейса и поведения): json
            Файл настроек игровых договоренностей: json, индивидуален для каждого игрока.
            Файл базы данных профилей (игроков-людей). json.
            Файлы картинок профилей
    
        Если настроек нет - создаем их из дефолтных
        Настройки едины для всех игроков, но разные для разных пользователей системы (т.к. в папке профиля хранятся)
        Игровые договоренность свои собственные у каждого игрока, т.е. при переключении игроков они подгружаются из профиля игрока,
            если нет - создаются на основе дефолтных.
    
        Что где храним:
            - параметры: в корне папки параметров
            - БД профилей: в корне папки параметров
            - игровые договоренности: в папке профиля игрока
            - сохранения игры: в папке профиля игрока /save
            - аватарка: в папке профиля игрока

Пользователи, авторизация, регистрация. Настройки профиля.
    Пользователи храняться как локально, так и на игровом сервере, но сейчас пока про локальных.
    Хранение на игровом сервере и синхронизацию локального профиля с удаленным буду делать потом, при разработке сервера.
    Компьютерные игроки только локальные и стандартно. Будет массив этих игроков, сейчас там только список имен - преобразую
        в словарь. Поля uid, name, статистика ). Аватарка грузится по uid (т.е. ее имя совпадает с uid), потом переименовать их.
        БД компьютерных игроков храним в проекте игры, в папке configs.
        Возможно когда-нибудь сделаю их добавление и редактирование... Но пока не планирую - зачем оно?
    Локальная база человеков такая: поля: uid, name (видимое имя), login, поля статистики. Аватарка грузится по uid, т.е. при
        добавлении она сохраняется с именем uid, но имя файла храним отдельным полем, потому что расширение может меняться.
    Пароль для локальной игры не предусмотрен, а при включении синхронизации с сервером придется его задать. Он будет храниться только на сервере,
        а проверяться только при авторизации на сервере и включении синхронизации с сервером.
    Логин должен быть уникальным (в пределах локальной БД тоже). При включении синхронизации с сервером проверяем на уникальность еще и там!
        Логин можно менять без проблем, т.к. ключем уникальности игрока для внутренних механизмов идентификации будет uid.
    При запуске приложения, если игроков нет, автоматически создается новый профиль без лишних вопросов.
        При желании потом его можно будет настроить в настройках профиля.
