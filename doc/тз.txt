Настройки
        
Пользователи, авторизация, регистрация. Настройки профиля.
    Пользователи храняться как локально, так и на игровом сервере, но сейчас пока про локальных.
    Хранение на игровом сервере и синхронизацию локального профиля с удаленным буду делать потом, при разработке сервера.
    Компьютерные игроки только локальные и стандартно. Только список имен. Аватарка по имени. Отдельно будет БД с их статистикой,
        храним в общей папке настроек. Ключ в ней - имя игрока. Наполнятся будет по мере того, как будет появляться статистика.
    Локальная база человеков такая: поля: uid, name (видимое имя), login, avatar, поля статистики. Аватарка сохраняется по имени файла,
        которое храним в одном из полей профиля.
    Пароль для локальной игры не предусмотрен, а при включении синхронизации с сервером придется его задать. Он будет храниться только на сервере,
        и проверяться только при авторизации на сервере и включении синхронизации с сервером.
    Логин должен быть уникальным (в пределах локальной БД тоже). При включении синхронизации с сервером проверяем на уникальность еще и там!
        Логин можно менять без проблем, т.к. ключем уникальности игрока для внутренних механизмов идентификации будет uid.
        Но при смене его так же надо проверить на уникальность.
    При запуске приложения, если игроков нет, автоматически создается новый профиль без лишних вопросов.
        При желании потом его можно будет настроить в настройках профиля.
        uid: генерим guid
        name: Фигню какую-то придумать, чтоб было веселей ) Может как сейчас - выбирать из списка.
        login: слепить из названия машины и имени пользователя 
        картинки нет. Сделать картинку "no image" и грузить ее, когда нет.
        картинки пользователей хранить в папке профиля этого пользователя (см. выше про хранение настроек).
    Создание нового игрока
        login, name ввести обязательно.
        Выбор картинки. Нужно сделать такую формочку с обрезкой / масштабированием до нужного размера (180 х 180).

Интерфейс
    1. Значкам мастей, которые показывают козырь, когда розданы все карты, нужно добавить какой-то свой фон (прям в png),
        потому что на фоне другого бэкграунда их может стать не видно. Или сам объект Lear сделать со своим фиксированным
        цветом фона (что сделать проще и удобнее - если что - его и подобрать быстро можно).
    2. Сделать свой диалог редактирования картинки. Обрезать как показал юзер, масштабирование, как захочет.
        Открывать картинку в квадратике 180 на 180 в исходном ее размере (т.е. так, чтоб был виден тот кусок, который влез).
        Давать юзеру изменять масштаб при прокрутке колесиком мышки и перетаскивать на тот участок, что он хочет видеть.
        И показать пару опций масштабирования - режим aspectRatioMode: метод сохранения масштаба и transformMode: метод фильтрации.
    *3. Добавить подсказки к некоторым параметрам (в хинтах) в окне договоренностей. В премиях/штрафах расписать, как она считается.
    4. Сделать меню хэлп и написать сам хэлп. Там будет описание игры, правила, описание параметров договоренностей,
        что как считается и т.п. Сама форма - просто большое текстовое поле с html-ным описанием.
        В окне настройки договоренностей в заголовке добавить кнопку с вопросиком и открывать по ней хэлп (Может быть еще
            скроллить на раздел с описанием договоренностей).
    +5. Добавить тем оформления.
    6. Поискать наборы колод более высокого качества.
       
Игровой процесс
    1. Сделать механизм ведения статистики партии по игрокам (не хранить ее, просто показать в конце игры):
        суммарная оценка всех раздач за игру (оценка розданных карт в каждой партии, что-то типа сколько хороших, сколько полхих с учетом прикрытых неприкрытых
            и т.д.)
        кол-во джокеров
        кол-во проигранных игр с джокером на руках (пока непонятно куда, наверное в общую оценку)
        кол-во рискованных конов (придумать, как оценивать; сразу показатель: ненулевой заказ на темных партиях и
            любой заказ в темную в обычных партиях +1 к рискованным)
        оценка мизеров: кол-во взяток на мизере, кол-во партий с 0 взяток, общий балл
        оценка золотых: кол-во взяток на золотых, кол-во партий с 0 взяток, общий балл
        оценка темных: всего темных партий, кол-во заказанных взяток, кол-во сыгранных, кол-во переборов, кол-во недоборов,
            общее кол-во несыгранных, кол-во партий в темную в обычных партиях
        общее соотношение заказов к взятому
        общее соотношение выигранных к проигранным
        кол-во сыгранных конов
        кол-во проигранных конов (это когда не взял свое)
        кол-во переборов
        кол-во недоборов
        Что-то из этого буду выводить в %
        Игровое время партии
    2. Переработать финальное окно в конце игры.
        Показывать общие итоги игры, которые сейчас, возможно только сдвинуть их куда-то (и может сделать меньше), чтоб мето освободить,
        можно переработать их вид.
        Сделать показ статистики из п.1. Сделать грид, примерно как таблица записи игры (можно даже взять из него цвета и пр.), только с родной шапкой от грида.
        Оставить кнопку показа таблицы записи игры на окне итогов игры, таблицу записи игры в конце не показывать автоматически.
